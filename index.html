vim实用技巧-未完待续

  • 删除
    • dd #删除一行
    • x #删除一个字符
    • dG #删除当前行到最后一行
    • :g/^$/d #删除文件中所有空行
    • :%s/^(.*)n1/1$/ #删除重复行
    • :%s/^\s*//g# #删除行首多个空格
    • :%s/\s+$//g #删除行尾多个空格
    • dw #删除一个单词
  • 移动
    • gg #移动到第一行
    • G #移动到最后一行
    • 0 #移动到最首
    • $ #移动到行尾
    • w #移动到下一个单词的起始位置
    • ^ #移动到当前行的非空行行首
    • nG #移动到第n行
    • h #左移
    • j #下移
    • k #上移
    • l #右移
    • ctrl + f #向下滚动一屏
    • ctrl + b #向上滚动一屏
  • 显示
    • :set nu #显示行号
    • :set nonu #取消显示行号
  • 撤消
    • u #撤消上个命令的修改
  • 插入
    • i #进入编辑模式
    • o #在当前行的下一行进行插入
    • yy + p #复制当前行并粘贴到下一行

MAC下三步搭建自己的hexo博客

Git安装

  • brew install git
  • 注册github，并成功使用，后续配置文件中要使用这个git仓库，所有发布的博客是托管在git服务器上的

Node.js安装

  • brew install node
  • 使用以下命令验证是否安装成功
    node -v;
    npm -v

Hexo安装和发布博客

  • npm install hexo -g #-g表示全局安装, npm默认为当前项目安装
  • hexo init xxx #新建博客目录，并且会在该目录下自动生成 _config.yml, - package.json文件和scaffolds, source, themes子目录
    • _config.yml为全局配置文件非常重要，配置博客的github地址，博客签名等信息
    • themes 为博客主题目录
    • source为本地存放博客的文件夹
    • package.json 应用程序数据，指明hexo的版本等信息
    • 其他目录和文件没研究过，但最好不要动
  • hexo g #根据当前目录下文件生成静态网页
  • npm install #初始化依赖
  • hexo s #启动服务器，可在本地http://0.0.0.0:4000/看到博客框架-

  • hexo n ‘我的第一篇文章’, 创建成功会有如下提示：INFO Created: ~/hexo/source/_posts/我的第一篇文章.md，博客内容的书写使用markdown语法，博客内容如下:

  • 找到对应下目录的文章并编辑，编辑好文章内容后依次执行如下命令就可以在 http://yourname.github.io/看到博客主页了

    • hexo g #生成
    • hexo s #本地预览
    • hexo d #发布到github
