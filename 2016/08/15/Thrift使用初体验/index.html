Thrift使用初体验

  • 简单介绍 Apache Thrift是一个facebook建立的RPC框架，现在是一个Apache的顶级项目。Thrift允许通过一个跨语言的定义文件的方式定义数据类型和服务接口，这个文件作为RPC客户端和服务器通信的标准
  • 安装 thrift
    • 下载: wget http://apache.mirror.iweb.ca/thrift/0.9.2/thrift-0.9.2.tar.gz
    • 解压: tar -xvzf thrift-0.9.0.tar.gz
    • 配置: cd thrift-0.9.2 然后 sudo ./configure
      • 可能遇到的报错:configure: error: Bison version 2.5 or higher must be installed on the system!
      • 如何解决:brew install bison, brew link bison –force
    • 编译: sudo make
    • 安装: sudo make install
    • 验证是否安装成功: thrift –help

  • 使用Thrift

    • 编写thrift文件，并命名为hello.thrift，详细语法可参考官网，如何定义了一个名为Hello的thrift服务，服务中包含一个方法 helloString，方法有返回值类型，方法名，参数列表

- 编译thrift文件，使用安装好的thrift编译器编译user.thrift生成python，java，c++代码
    - thrift --gen java hello.thrift
    - thrift --gen py hello.thrift
    - 生成结果: gen-java，gen-py目录，gen-java目录下有Hello.java文件
- python使用上述编译好的python代码还需要添加python所需要的第三方依赖，使用pip istall thrift；同样的,java使用上述编译好的java代码需要添加maven依赖
  • 编写python服务端
    • 编写python服务端,需要将上面生成的python代码拷贝到你的项目里面，或者通过包引用的方式将目录添加进来，再引入到项目里面，java也是一样。将生成的python代码通过引用的方式引入到python环境。只需要在生成的gen-py目录下编写一个setup.py文件即可。然后执行 python setup.py install 命令。文件内容如下，使用import hello 验证是否安装成功
       #!/usr/bin/env python
       # -*- coding: utf-8 -*-

       from setuptools import find_packages
       from setuptools import setup
       install_requires = ['']
       setup(
             name='hello',
             version='0.1.1',
             author='xiaocui',
             author_email='xiaocui@zhihu.com',
             description='thrift-demo',
             license='PRIVATE',
             packages=find_packages(),
             install_requires=install_requires,
            )


- 编写pyhon服务端代码，服务代码要实现thrift文件里的接口方法，方法名，参数类型，返回值类型要一致

     #!/usr/bin/env python
     # encoding: utf-8
     from hello import Hello
     from thrift.transport import TSocket
     from thrift.transport import TTransport
     from thrift.protocol import TBinaryProtocol
     from thrift.server import TServer

     class HelloHandler:
         def __init__(self):
             pass
     def helloString(self, word):
         ret = "Received: " + word
         return ret

     handler = HelloHandler()
     processor = Hello.Processor(handler)
     transport = TSocket.TServerSocket("127.0.0.1", 8989)
     tfactory = TTransport.TBufferedTransportFactory()
     pfactory = TBinaryProtocol.TBinaryProtocolFactory()
     server = TServer.TSimpleServer(processor, transport, tfactory, pfactory)
     print "Starting thrift server in python..."
     server.serve()
     print "done!"

  • python hello-thrift-server.py启动thrift服务，输出结果: Starting thrift server in python... 表示在等待请求

    • 编写java客户端调用python服务

    • 如何使用java调用接python server中的helloString方法呢，或者说调用python thrift服务呢？这里需要进一步理解thrift文件的意义，无论python调用java还是java调用python实际上都是在使用对方的具体方法的方法体，而thrift文件就是一个中间桥梁，或者说是一个公共接口定义文件，接口文件里定义的是方法名及其返回值，参数列表等。在编写python服务时需要去实现你在thrift文件里定义的方法，启动服务实际上是启动了一个socket在固定端口上不断的监听有没有请求，相应的请求实际上是在调用你编写的方法体，这个请求可以是其他语言如java发出的请求。所以当你在编写client端时，可以是java语言，GO语言等。但是无论哪种语言，你必须在编写client端代码时，引入thrift编译的文件，如果你使用java语言则需要引入上述编译好的Hello.java文件，才能使用thrift提供的java接口，且这个接口的实现体已经被python服务实现。总结一句话:服务端负责实现thrift文件里的接口，客户端负责调用服务端实现的接口，且一种语言编写的服务端可以被多种语言的客户端调用。
